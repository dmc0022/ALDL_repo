# Sensor Board FPGA System

This repository contains the Verilog/SystemVerilog source files, display assets, and Quartus project files for the **sensor board FPGA interface system**. The design targets the DE10-Lite / Intel MAX 10 FPGA platform and integrates an LCD touch interface, CAP1188 capacitive touch sensing, SPI display control, I2C communication, and several demo applications such as Breakout, keypad input, GIF/image rendering, and temperature display.

The project is organized around a top-level system module, `aldl_system_top.v`, which connects the display pipeline, touch controller, sensor interfaces, and application renderers into one FPGA design.

## Repository Contents

### Top-Level Project Files

| File | Description |
|---|---|
| `aldl_system_top.v` | Main FPGA top-level module. Connects the LCD, touch interface, CAP1188, temperature display, and app selection logic. |
| `aldl_system_top.qpf` | Quartus project file. |
| `aldl_system_top.qsf` | Quartus settings and pin assignment file. |

## Display and LCD Driver Files

| File | Description |
|---|---|
| `LCD_driver_core_shared_touch.v` | Shared LCD/touch driver logic used by the system. |
| `tft_ili9341.v` | High-level ILI9341 TFT LCD controller. Handles initialization and full-frame pixel writes. |
| `tft_ili9341_spi.v` | Low-level SPI interface for sending commands and pixel data to the LCD. |
| `spi_master_byte.v` | SPI byte transmitter used by the display driver. |
| `font4x7.v` | Small bitmap font used for text rendering on the LCD. |

The LCD system uses an ILI9341 TFT display driven over SPI. Pixel data is generated by the active renderer module and streamed to the display through the TFT driver.

---

## Touch and I2C Files

| File | Description |
|---|---|
| `i2c_touch_hub.v` | Handles touch-related I2C communication and routes touch data into the application system. |
| `i2c_master.sv` | SystemVerilog I2C master module. |
| `gI2C_low_level_tx_rx.v` | Low-level I2C transmit/receive module used for I2C peripheral communication. |
| `cycle_timer.sv` | Timing helper used by the I2C master. |
| `new_CAP1188.v` | CAP1188 capacitive touch sensor interface module. |

The project uses I2C to communicate with external sensor/touch devices. The CAP1188 provides capacitive touch input, while the LCD touch controller provides screen coordinate data for app selection and interaction.

## Application Renderer Files

| File | Description |
|---|---|
| `home_renderer.v` | Draws the home screen and app icons. |
| `breakout_renderer.v` | Draws the Breakout game screen, including paddle, ball, bricks, score, and game-over UI. |
| `breakout_game.v` | Contains the main Breakout game logic. |
| `keypad_renderer.v` | Draws the on-screen keypad interface. |
| `keypad_touch_decode.v` | Converts touch coordinates into keypad button presses. |
| `gif_renderer.v` | Draws the GIF/image demo screen. |
| `temperature_renderer.v` | Draws the temperature display screen. |
| `image_rom.v` | ROM module used for image or sprite data. |

Each renderer is responsible for generating RGB565 pixel data for a specific application screen. The top-level module selects which renderer is active based on the current app state and touch input.

## System Overview

The FPGA system operates as a small touch-based application interface. On startup, the LCD displays a home screen with multiple app icons. The user can select an application using the LCD touch interface. Depending on the selected app, the system switches to one of several renderer modules.

Main applications include:

- **Home Screen**  
  Displays app icons and handles navigation.

- **Breakout Game**  
  A simple FPGA-rendered Breakout game using sprite ROMs and real-time game logic.

- **Keypad App**  
  Displays a touch keypad and decodes touch locations into keypad values.

- **GIF/Image Demo**  
  Displays image-based graphics from ROM.

- **Temperature Display**  
  Displays temperature/sensor-related information on the LCD and/or seven-segment displays.

- **CAP1188 Sensor Interface**  
  Reads capacitive touch sensor input through I2C.

---

## Hardware Used

This project is intended for an FPGA-based sensor board system using:

- DE10-Lite FPGA board / Intel MAX 10 FPGA
- ILI9341 320×240 TFT LCD display
- FT6336G capacitive touch controller
- CAP1188 capacitive touch sensor
- SPI interface for LCD communication
- I2C interface for touch and sensor communication
- Seven-segment displays for debug output
- External sprite/image memory initialized from `.hex` files

## Build Instructions

1. Open **Quartus Prime**.
2. Open the project file:

   ```text
   aldl_system_top.qpf
   ```

3. Confirm that all Verilog/SystemVerilog and `.hex` files are included in the project.
4. Confirm that the pin assignments in `aldl_system_top.qsf` match the FPGA board and connected hardware.
5. Compile the project.
6. Program the FPGA using the generated `.sof` file.

---

## Notes for Development

- The main integration point is `aldl_system_top.v`.
- Renderer modules should output RGB565 pixel data.
- The LCD driver expects continuous frame data for the full display.
- I2C devices must have the correct device address and pull-up resistors on SDA/SCL.
- `.hex` sprite files must remain in the project directory so Quartus can initialize ROM contents correctly.
- Seven-segment outputs are useful for debugging I2C communication and sensor values.

## Project Purpose

This repository was created as part of an FPGA-based sensor board and display interface project. The goal is to demonstrate how multiple hardware peripherals can be integrated into a single FPGA system using Verilog/SystemVerilog, including SPI display control, I2C sensor communication, touch input, image rendering, game logic, and real-time user interaction.
